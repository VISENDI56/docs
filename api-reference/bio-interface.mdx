## Somatic authentication

Authenticate users using somatic traits (posture, location, stillness) as cryptographic factors.

### Endpoint

```
POST /api/bio/authenticate
```

### Request body

```json
{
  "somatic_data": {
    "posture": 85.0,
    "location": {
      "lat": 1.2921,
      "lng": 36.8219
    },
    "stillness": 78.5,
    "heart_rate": 72.0,
    "skin_temperature": 36.5
  },
  "user_id": "OPERATOR_001",
  "context": "clinical_access"
}
```

### Request fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `somatic_data.posture` | float | Yes | Body posture angle (0-180 degrees, optimal: 90) |
| `somatic_data.location` | object | Yes | GPS coordinates {lat, lng} |
| `somatic_data.stillness` | float | Yes | Movement detection (0-100, higher = more still) |
| `somatic_data.heart_rate` | float | No | Heart rate in BPM |
| `somatic_data.skin_temperature` | float | No | Skin temperature in Celsius |
| `user_id` | string | Yes | Operator identifier |
| `context` | string | No | Authentication context (default: "clinical_access") |

### Response

```json
{
  "authenticated": true,
  "confidence": 0.892,
  "token": "bio_token_847392",
  "biometric_signature": "{\"user_id\":\"OPERATOR_001\",\"timestamp\":\"2025-12-19T20:00:00\",\"somatic_hash\":12345678,\"confidence\":0.892}",
  "timestamp": "2025-12-19T20:00:00.000Z"
}
```

### Response fields

| Field | Type | Description |
|-------|------|-------------|
| `authenticated` | boolean | Authentication success (confidence > 0.8) |
| `confidence` | float | Authentication confidence score (0-1) |
| `token` | string | Biometric session token (if authenticated) |
| `biometric_signature` | string | Cryptographic signature of somatic data |
| `timestamp` | string | ISO 8601 timestamp |

### Confidence calculation

```python
posture_score = min(1.0, posture / 90.0)  # Optimal at 90 degrees
stillness_score = stillness / 100.0
location_accuracy = 1.0  # GPS accuracy

confidence = (posture_score * 0.4 + stillness_score * 0.4 + location_accuracy * 0.2)
authenticated = confidence > 0.8
```

### Example: Python

```python
import requests

response = requests.post(
    "http://localhost:8000/api/bio/authenticate",
    json={
        "somatic_data": {
            "posture": 85.0,
            "location": {"lat": 1.2921, "lng": 36.8219},
            "stillness": 78.5,
            "heart_rate": 72.0,
            "skin_temperature": 36.5
        },
        "user_id": "OPERATOR_001",
        "context": "clinical_access"
    }
)

result = response.json()

if result["authenticated"]:
    print(f"✅ Authenticated - Confidence: {result['confidence']}")
    print(f"Token: {result['token']}")
else:
    print(f"❌ Authentication failed - Confidence: {result['confidence']}")
```

## Stress assessment

Real-time stress assessment for Silent Flux information regulation.

### Endpoint

```
POST /api/bio/stress-assess
```

### Request body

```json
{
  "posture": 85.0,
  "location": {
    "lat": 1.2921,
    "lng": 36.8219
  },
  "stillness": 78.5,
  "heart_rate": 95.0,
  "skin_temperature": 37.2
}
```

### Response

```json
{
  "stress_level": 0.687,
  "recommendations": [
    "Activate Silent Mode - reduce information flow",
    "Minimize alerts and notifications",
    "Enable simplified interface",
    "Schedule follow-up monitoring"
  ],
  "monitoring_required": true
}
```

### Response fields

| Field | Type | Description |
|-------|------|-------------|
| `stress_level` | float | Stress level (0-1 scale) |
| `recommendations` | array | Recommended actions based on stress level |
| `monitoring_required` | boolean | Whether continuous monitoring is needed |

### Stress levels

| Level | Range | Silent Flux Mode | Description |
|-------|-------|------------------|-------------|
| **Low** | 0.0-0.3 | FULL | Normal operation, all information shown |
| **Moderate** | 0.3-0.5 | FULL | Monitor closely, no throttling yet |
| **Elevated** | 0.5-0.7 | FILTERED | Filter low-priority information |
| **High** | 0.7-0.8 | CRITICAL_ONLY | Show only critical alerts |
| **Critical** | 0.8-1.0 | SILENT | Emergency silence mode |

### Stress calculation

```python
heart_stress = (heart_rate - 60) / 40
temp_stress = abs(skin_temperature - 36.5) / 2
posture_stress = abs(posture - 90) / 90
stillness_stress = (100 - stillness) / 100

stress_level = (heart_stress + temp_stress + posture_stress + stillness_stress) / 4
```

### Example: Python

```python
import requests

response = requests.post(
    "http://localhost:8000/api/bio/stress-assess",
    json={
        "posture": 85.0,
        "location": {"lat": 1.2921, "lng": 36.8219},
        "stillness": 78.5,
        "heart_rate": 95.0,
        "skin_temperature": 37.2
    }
)

result = response.json()

print(f"Stress Level: {result['stress_level']:.1%}")
print(f"Monitoring Required: {result['monitoring_required']}")
print("\nRecommendations:")
for rec in result['recommendations']:
    print(f"  - {rec}")
```

## Integration with Acorn Protocol

The Bio-Interface API integrates with the Acorn Protocol for hardware-level somatic authentication:

```python
from hardware.acorn_protocol import AcornProtocol
import requests

# Initialize Acorn Protocol
acorn = AcornProtocol(operator_id="OPERATOR_001")

# Capture somatic signature from hardware sensors
signature = acorn.capture_somatic_signature(
    tilt_x=5.2,
    tilt_y=-3.1,
    latitude=1.2921,
    longitude=36.8219,
    movement_magnitude=0.12
)

# Authenticate via Bio-Interface API
response = requests.post(
    "http://localhost:8000/api/bio/authenticate",
    json={
        "somatic_data": {
            "posture": signature.posture,
            "location": {"lat": signature.latitude, "lng": signature.longitude},
            "stillness": signature.stillness
        },
        "user_id": "OPERATOR_001"
    }
)

if response.json()["authenticated"]:
    print("✅ Somatic authentication successful")
```

## Integration with Silent Flux

The Bio-Interface API provides stress data for Silent Flux information regulation:

```python
from edge_node.frenasa_engine.silent_flux import SilentFlux
import requests

# Get stress assessment
response = requests.post(
    "http://localhost:8000/api/bio/stress-assess",
    json={
        "posture": 85.0,
        "location": {"lat": 1.2921, "lng": 36.8219},
        "stillness": 78.5,
        "heart_rate": 95.0,
        "skin_temperature": 37.2
    }
)

stress_data = response.json()

# Initialize Silent Flux
flux = SilentFlux()

# Set output mode based on stress level
if stress_data["stress_level"] > 0.8:
    flux.set_output_mode("SILENT")
elif stress_data["stress_level"] > 0.6:
    flux.set_output_mode("CRITICAL_ONLY")
elif stress_data["stress_level"] > 0.3:
    flux.set_output_mode("FILTERED")
else:
    flux.set_output_mode("FULL")

# Filter AI inferences based on stress
filtered_alerts = flux.filter_inferences(all_alerts)
```

## Duress detection

The Bio-Interface API detects duress scenarios (coercion, panic access):

### Duress indicators

- **Abnormal posture** - Significant deviation from baseline
- **Elevated heart rate** - >100 BPM sustained
- **High skin temperature** - >37.5°C
- **Excessive movement** - Stillness <30
- **Location anomaly** - Unexpected geographic location

### Duress response

When duress is detected (confidence < 0.5):
1. Require additional verification
2. Limit access to sensitive operations
3. Alert security monitoring systems
4. Create audit trail of duress indicators
