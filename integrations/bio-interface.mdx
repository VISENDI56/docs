## Overview

The Bio-Interface is iLuminara's REST API for mobile health applications, enabling Community Health Volunteers (CHVs) to submit health data that automatically integrates with the Golden Thread data fusion engine.

## API endpoints

### Submit health observation

```
POST /api/v1/observations
```

Submit a health observation from a CHV.

**Request:**

```json
{
  "chv_id": "CHV_AMINA_HASSAN",
  "patient_id": "PAT_12345",
  "location": {
    "lat": 0.0512,
    "lng": 40.3129,
    "name": "Dadaab Refugee Camp"
  },
  "timestamp": "2025-12-24T10:30:00Z",
  "symptoms": ["fever", "diarrhea", "vomiting"],
  "severity": 8,
  "notes": "Patient reports severe dehydration",
  "consent_token": "CONSENT_789"
}
```

**Response:**

```json
{
  "status": "success",
  "observation_id": "OBS_456",
  "verification_score": 0.0,
  "verification_status": "PENDING_EMR",
  "golden_thread_id": "GT_789",
  "recommendations": [
    "IMMEDIATE: Suspected cholera. Start ORS immediately.",
    "Isolate patient to prevent spread",
    "Notify district health officer"
  ],
  "compliance": {
    "sovereignty_validated": true,
    "consent_validated": true,
    "jurisdiction": "KDPA_KE"
  }
}
```

### Submit voice alert

```
POST /api/v1/voice-alerts
```

Submit a voice recording for transcription and analysis.

**Request:**

```bash
curl -X POST https://api.iluminara.health/v1/voice-alerts \
  -H "Authorization: Bearer $CHV_TOKEN" \
  -H "Content-Type: audio/wav" \
  -F "audio=@alert.wav" \
  -F "chv_id=CHV_AMINA_HASSAN" \
  -F "location_lat=0.0512" \
  -F "location_lng=40.3129"
```

**Response:**

```json
{
  "status": "success",
  "transcription": "Patient reporting severe watery diarrhea and vomiting",
  "language_detected": "swahili",
  "symptoms": ["diarrhea", "vomiting", "dehydration"],
  "severity": 9,
  "alert_level": "CRITICAL",
  "observation_id": "OBS_457",
  "processing_time_ms": 4200
}
```

## Authentication

### CHV token

```bash
# Obtain CHV token
curl -X POST https://api.iluminara.health/v1/auth/token \
  -H "Content-Type: application/json" \
  -d '{
    "chv_id": "CHV_AMINA_HASSAN",
    "password": "secure_password",
    "device_id": "DEVICE_001"
  }'
```

**Response:**

```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "refresh_token": "refresh_token_here"
}
```

### Use token

```bash
curl -X POST https://api.iluminara.health/v1/observations \
  -H "Authorization: Bearer $ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d @observation.json
```

## Mobile SDK

### Android (Kotlin)

```kotlin
import com.iluminara.sdk.ILuminaraClient
import com.iluminara.sdk.models.Observation

// Initialize client
val client = ILuminaraClient(
    apiKey = "your_api_key",
    baseUrl = "https://api.iluminara.health"
)

// Submit observation
val observation = Observation(
    chvId = "CHV_AMINA_HASSAN",
    patientId = "PAT_12345",
    location = Location(lat = 0.0512, lng = 40.3129),
    symptoms = listOf("fever", "diarrhea"),
    severity = 8,
    consentToken = "CONSENT_789"
)

client.submitObservation(observation) { result ->
    when (result) {
        is Success -> {
            println("Observation ID: ${result.data.observationId}")
            println("Verification: ${result.data.verificationStatus}")
        }
        is Error -> {
            println("Error: ${result.error.message}")
        }
    }
}
```

### iOS (Swift)

```swift
import ILuminaraSDK

// Initialize client
let client = ILuminaraClient(
    apiKey: "your_api_key",
    baseURL: "https://api.iluminara.health"
)

// Submit observation
let observation = Observation(
    chvId: "CHV_AMINA_HASSAN",
    patientId: "PAT_12345",
    location: Location(lat: 0.0512, lng: 40.3129),
    symptoms: ["fever", "diarrhea"],
    severity: 8,
    consentToken: "CONSENT_789"
)

client.submitObservation(observation) { result in
    switch result {
    case .success(let response):
        print("Observation ID: \(response.observationId)")

    case .failure(let error):
        print("Error: \(error.localizedDescription)")
    }
}
```

### React Native (JavaScript)

```javascript
import { ILuminaraClient } from '@iluminara/sdk';

// Initialize client
const client = new ILuminaraClient({
  apiKey: 'your_api_key',
  baseURL: 'https://api.iluminara.health'
});

// Submit observation
const observation = {
  chv_id: 'CHV_AMINA_HASSAN',
  patient_id: 'PAT_12345',
  location: { lat: 0.0512, lng: 40.3129 },
  symptoms: ['fever', 'diarrhea'],
  severity: 8,
  consent_token: 'CONSENT_789'
};

client.submitObservation(observation)
  .then(response => {
    console.log('Observation ID:', response.observation_id);
    console.log('Verification:', response.verification_status);
  })
  .catch(error => {
    console.error('Error:', error.message);
  });
```

## Offline support

### Queue observations

```javascript
// Queue observation when offline
if (!navigator.onLine) {
  await client.queueObservation(observation);
  console.log('Observation queued for sync');
}

// Auto-sync when online
window.addEventListener('online', async () => {
  const result = await client.syncQueue();
  console.log(`Synced ${result.synced_count} observations`);
});
```

### Local storage

```javascript
// Store observations locally
const localDB = new LocalDatabase();

await localDB.save('observations', observation);

// Sync when connectivity returns
const pendingObservations = await localDB.getAll('observations');
await client.syncObservations(pendingObservations);
```

## Golden Thread integration

Every observation submitted via Bio-Interface becomes a CBS (Community-Based Surveillance) signal:

```python
from edge_node.sync_protocol.golden_thread import GoldenThread

gt = GoldenThread()

# Bio-Interface observation becomes CBS signal
fused = gt.fuse_data_streams(
    cbs_signal={
        'observation_id': 'OBS_456',
        'location': 'Dadaab',
        'symptom': 'fever',
        'timestamp': '2025-12-24T10:30:00Z',
        'source': 'CHV_AMINA_HASSAN'
    },
    patient_id='PAT_12345'
)

# Check verification score
print(f"Verification: {fused.verification_score}")
# 0.0 (PENDING_EMR) - waiting for EMR confirmation
```

```python
from governance_kernel.vector_ledger import SovereignGuardrail
```

```
Rate Limit: 100 requests/minute per CHV
Burst Limit: 10 requests/second
```

**Headers:**

```
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1640347200
```

## Error codes

| Code | HTTP Status | Description |
|------|-------------|-------------|
| `invalid_token` | 401 | Authentication token invalid or expired |
| `missing_consent` | 403 | Consent token required but not provided |
| `sovereignty_violation` | 403 | Action violates data sovereignty rules |
| `invalid_location` | 400 | GPS coordinates invalid or out of range |
| `rate_limit_exceeded` | 429 | Too many requests |
| `offline_queue_full` | 507 | Local storage quota exceeded |
