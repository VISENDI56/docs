## Overview

The Bio-Interface provides a REST API for mobile health applications to integrate with iLuminara-Core, using the Golden Thread protocol to fuse data streams from Community-Based Surveillance (CBS) and Electronic Medical Records (EMR).

## API endpoints

### 1. Voice processing

Process voice alerts from CHVs and convert to structured health data.

**Endpoint:** `POST /process-voice`

**Headers:**
```
Content-Type: audio/wav
Authorization: Bearer <token>
```

**Query parameters:**
- `language` (optional): Language of audio (default: \"swahili\")
- `lat` (optional): Latitude coordinate
- `lng` (optional): Longitude coordinate

**Response:**
```json
{
  \"status\": \"success\",
  \"timestamp\": \"2025-12-19T20:00:00.000Z\",
  \"transcription\": \"Patient reporting severe diarrhea and vomiting\",
  \"symptoms\": [\"diarrhea\", \"vomiting\"],
  \"severity\": 9,
  \"location\": {\"lat\": 0.0512, \"lng\": 40.3129},
  \"alert_level\": \"CRITICAL\",
  \"verification_score\": 0.8,
  \"golden_thread_id\": \"GT_12345\"
}
```

### 2. Symptom reporting

Report symptoms directly from mobile apps.

**Endpoint:** `POST /report-symptoms`

**Request:**
```json
{
  \"patient_id\": \"PAT_12345\",
  \"symptoms\": [\"fever\", \"cough\", \"fatigue\"],
  \"severity\": 7,
  \"location\": {\"lat\": 0.0512, \"lng\": 40.3129},
  \"timestamp\": \"2025-12-19T20:00:00.000Z\",
  \"source\": \"patient_self_report\"
}
```

**Response:**
```json
{
  \"status\": \"success\",
  \"report_id\": \"CBS_67890\",
  \"verification_score\": 0.0,
  \"verification_status\": \"UNVERIFIED\",
  \"recommendations\": [
    \"Seek medical attention if symptoms worsen\",
    \"Monitor temperature every 4 hours\"
  ],
  \"golden_thread_id\": \"GT_12346\"
}
```

### 3. Outbreak prediction

Predict outbreak risk based on symptoms and location.

**Endpoint:** `POST /predict`

**Request:**
```json
{
  \"location\": {\"lat\": 0.4221, \"lng\": 40.2255},
  \"symptoms\": [\"diarrhea\", \"vomiting\"],
  \"population_size\": 80000,
  \"time_window_days\": 7
}
```

**Response:**
```json
{
  \"status\": \"success\",
  \"risk_score\": 0.87,
  \"risk_level\": \"HIGH\",
  \"predicted_cases\": 156,
  \"confidence_interval\": [120, 195],
  \"disease_likelihood\": {
    \"cholera\": 0.92,
    \"typhoid\": 0.45,
    \"dysentery\": 0.38
  },
  \"recommendations\": [
    \"Activate emergency response protocol\",
```

## Golden Thread integration

### CBS signal creation

When a mobile app reports symptoms, it creates a CBS (Community-Based Surveillance) signal:

```python
from edge_node.sync_protocol.golden_thread import GoldenThread

gt = GoldenThread()

# Create CBS signal from mobile report
cbs_signal = {
    'location': request.json['location'],
    'symptom': request.json['symptoms'][0],
    'timestamp': request.json['timestamp'],
    'source': 'mobile_app',
    'reporter_id': request.json['patient_id']
}

# Attempt to fuse with EMR data
fused = gt.fuse_data_streams(
    cbs_signal=cbs_signal,
    patient_id=request.json['patient_id']
)

# Check verification score
if fused.verification_score >= 0.8:
    status = \"CONFIRMED\"
elif fused.verification_score >= 0.5:
    status = \"PROBABLE\"
else:
    status = \"UNVERIFIED\"
```

### Verification logic

The Golden Thread verifies CBS signals against EMR records:

```
IF cbs.location == emr.location AND |cbs.timestamp - emr.timestamp| < 24h
    THEN verification_score = 1.0 (CONFIRMED)
ELSE IF cbs.location ≈ emr.location AND |cbs.timestamp - emr.timestamp| < 48h
    THEN verification_score = 0.7 (PROBABLE)
ELSE
    verification_score = 0.3 (UNVERIFIED)
```

## Mobile SDK

### Android SDK

```kotlin
// Initialize iLuminara SDK
val iluminara = ILuminaraSDK.Builder()
    .setApiUrl(\"https://api.iluminara.health\")
    .setApiKey(\"your-api-key\")
    .setJurisdiction(Jurisdiction.KDPA_KE)
    .build()

// Report symptoms
val symptoms = listOf(\"fever\", \"cough\", \"fatigue\")
val location = Location(0.0512, 40.3129)

iluminara.reportSymptoms(
    patientId = \"PAT_12345\",
    symptoms = symptoms,
    severity = 7,
    location = location
) { result ->
    when (result) {
        is Success -> {
            println(\"Report ID: ${result.data.reportId}\")
            println(\"Verification: ${result.data.verificationStatus}\")
        }
        is Error -> {
            println(\"Error: ${result.message}\")
        }
    }
}
```

### iOS SDK

```swift
// Initialize iLuminara SDK
let iluminara = ILuminaraSDK(
    apiUrl: \"https://api.iluminara.health\",
    apiKey: \"your-api-key\",
    jurisdiction: .KDPA_KE
)

// Report symptoms
let symptoms = [\"fever\", \"cough\", \"fatigue\"]
let location = Location(lat: 0.0512, lng: 40.3129)

iluminara.reportSymptoms(
    patientId: \"PAT_12345\",
    symptoms: symptoms,
    severity: 7,
    location: location
) { result in
    switch result {
    case .success(let response):
        print(\"Report ID: \\(response.reportId)\")
        print(\"Verification: \\(response.verificationStatus)\")
    case .failure(let error):
        print(\"Error: \\(error.localizedDescription)\")
    }
}
```

### React Native SDK

```javascript
import { ILuminaraSDK } from '@iluminara/react-native-sdk';

// Initialize SDK
const iluminara = new ILuminaraSDK({
  apiUrl: 'https://api.iluminara.health',
  apiKey: 'your-api-key',
  jurisdiction: 'KDPA_KE'
});

// Report symptoms
const reportSymptoms = async () => {
  try {
    const response = await iluminara.reportSymptoms({
      patientId: 'PAT_12345',
      symptoms: ['fever', 'cough', 'fatigue'],
      severity: 7,
      location: { lat: 0.0512, lng: 40.3129 }
    });
    
    console.log('Report ID:', response.reportId);
    console.log('Verification:', response.verificationStatus);
  } catch (error) {
    console.error('Error:', error.message);
  }
};
```

## Authentication

### API key authentication

```bash
curl -X POST https://api.iluminara.health/report-symptoms \\
  -H \"Authorization: Bearer your-api-key\" \\
  -H \"Content-Type: application/json\" \\
  -d '{
    \"patient_id\": \"PAT_12345\",
    \"symptoms\": [\"fever\", \"cough\"],
    \"severity\": 7,
    \"location\": {\"lat\": 0.0512, \"lng\": 40.3129}
  }'
```

### OAuth 2.0 authentication

```python
import requests

# Get access token
token_response = requests.post(
    'https://auth.iluminara.health/oauth/token',
    data={
        'grant_type': 'client_credentials',
        'client_id': 'your-client-id',
        'client_secret': 'your-client-secret'
    }
)

access_token = token_response.json()['access_token']

# Use access token
response = requests.post(
    'https://api.iluminara.health/report-symptoms',
    headers={'Authorization': f'Bearer {access_token}'},
    json={
        'patient_id': 'PAT_12345',
        'symptoms': ['fever', 'cough'],
        'severity': 7,
        'location': {'lat': 0.0512, 'lng': 40.3129}
    }
)
```

## Offline support

### Queue and sync

```javascript
// Queue report when offline
if (!navigator.onLine) {
  await iluminara.queueReport({
    patientId: 'PAT_12345',
    symptoms: ['fever', 'cough'],
    severity: 7,
    location: { lat: 0.0512, lng: 40.3129 }
  });
  
  console.log('Report queued for sync');
}

// Sync when online
window.addEventListener('online', async () => {
  const syncResult = await iluminara.syncQueuedReports();
  console.log(`Synced ${syncResult.count} reports`);
});
```
