## Overview

The Bio-Interface REST API enables seamless integration between mobile health applications and iLuminara-Core's Golden Thread data fusion engine. Designed for offline-first operation with automatic sync when connectivity returns.

## Authentication

All API requests require authentication using JWT tokens:

```bash
curl -X POST https://api.iluminara.health/v1/auth/login \\
  -H "Content-Type: application/json" \\
  -d '{
    "username": "chv_amina",
    "password": "secure_password",
    "device_id": "DEVICE_001"
  }'
```

Response:
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expires_in": 3600,
  "token_type": "Bearer"
}
```

## Core endpoints

### 1. Submit health observation

Submit a community-based surveillance (CBS) signal from a Community Health Volunteer (CHV).

**Endpoint:** `POST /v1/observations`

**Headers:**
```
Authorization: Bearer {access_token}
Content-Type: application/json
```

**Request body:**
```json
{
  "observation_id": "OBS-2025-001",
  "timestamp": "2025-12-28T22:00:00Z",
  "location": {
    "lat": 0.0512,
    "lng": 40.3129,
    "name": "Dadaab Refugee Camp"
  },
  "patient": {
    "patient_id": "PAT-12345",
    "age": 35,
    "gender": "F",
    "consent_token": "CONSENT-TOKEN-123"
  },
  "symptoms": [
    {
      "symptom": "diarrhea",
      "severity": 8,
      "duration_hours": 24
    },
    {
      "symptom": "vomiting",
      "severity": 7,
      "duration_hours": 12
    }
  ],
  "vital_signs": {
    "temperature_celsius": 38.5,
    "heart_rate_bpm": 95,
    "respiratory_rate": 22
  },
  "source": "CHV_VOICE_ALERT",
  "offline_queued": false
}
```

**Response:**
```json
{
  "status": "success",
  "observation_id": "OBS-2025-001",
  "fused_record_id": "FUSED-2025-001",
  "verification_score": 0.8,
  "alert_level": "HIGH",
  "recommendations": [
    "Suspected cholera - Start ORS immediately",
    "Isolate patient to prevent spread",
    "Notify district health officer"
  ],
  "golden_thread": {
    "cbs_signal_recorded": true,
    "emr_match_found": false,
    "idsr_notification_sent": true
  }
}
```

### 2. Voice processing

Process voice alerts from CHVs and convert to structured data.

**Endpoint:** `POST /v1/voice/process`

**Headers:**
```
Authorization: Bearer {access_token}
Content-Type: audio/wav
```

**Query parameters:**
- `language` (optional): Language of audio (default: "swahili")
- `lat` (optional): Latitude coordinate
- `lng` (optional): Longitude coordinate

**Request:**
```bash
curl -X POST "https://api.iluminara.health/v1/voice/process?language=swahili&lat=0.0512&lng=40.3129" \\
  -H "Authorization: Bearer {access_token}" \\
  -H "Content-Type: audio/wav" \\
  --data-binary @voice-alert.wav
```

**Response:**
```json
{
  "status": "success",
  "transcription": "Patient reporting severe watery diarrhea and vomiting",
  "language_detected": "swahili",
  "symptoms": ["diarrhea", "vomiting", "dehydration"],
  "severity": 9,
  "location": {
    "lat": 0.0512,
    "lng": 40.3129
  },
  "alert_level": "CRITICAL",
  "observation_id": "OBS-2025-002"
}
```

### 3. Outbreak prediction

Predict outbreak risk based on symptoms and location.

**Endpoint:** `POST /v1/predict/outbreak`

**Request body:**
```json
{
  "location": {
    "lat": 0.4221,
    "lng": 40.2255,
    "name": "Nairobi"
  },
  "symptoms": ["diarrhea", "vomiting"],
  "population": 100000,
  "time_window_days": 7
}
```

**Response:**
```json
{
  "status": "success",
  "risk_score": 0.87,
  "risk_level": "HIGH",
  "predicted_cases_72h": [18, 22, 27],
  "r_effective": 2.3,
  "z_score": 3.8,
  "confidence_interval": [0.75, 0.95],
  "recommendations": [
    "Increase surveillance in affected area",
    "Prepare emergency response teams",
    "Stock ORS and antibiotics"
  ]
}
```

### 4. Sync offline queue

Sync queued observations that were collected offline.

**Endpoint:** `POST /v1/sync/queue`

**Request body:**
```json
{
  "device_id": "DEVICE_001",
  "queued_observations": [
    {
      "observation_id": "OBS-OFFLINE-001",
      "timestamp": "2025-12-28T20:00:00Z",
      "location": {"lat": 0.0512, "lng": 40.3129},
      "symptoms": ["fever", "cough"]
    },
    {
      "observation_id": "OBS-OFFLINE-002",
      "timestamp": "2025-12-28T21:00:00Z",
      "location": {"lat": 0.0512, "lng": 40.3129},
      "symptoms": ["headache", "fatigue"]
    }
  ]
}
```

**Response:**
```json
{
  "status": "success",
  "synced_count": 2,
  "failed_count": 0,
  "results": [
    {
      "observation_id": "OBS-OFFLINE-001",
      "status": "synced",
      "fused_record_id": "FUSED-2025-003"
    },
    {
      "observation_id": "OBS-OFFLINE-002",
      "status": "synced",
      "fused_record_id": "FUSED-2025-004"
    }
  ]
}
```

### 5. Get patient history

Retrieve patient health history with Golden Thread verification.

**Endpoint:** `GET /v1/patients/{patient_id}/history`

**Query parameters:**
- `start_date` (optional): Start date (ISO 8601)
- `end_date` (optional): End date (ISO 8601)
- `include_cbs` (optional): Include CBS signals (default: true)
- `include_emr` (optional): Include EMR records (default: true)

**Response:**
```json
{
  "status": "success",
  "patient_id": "PAT-12345",
  "records": [
    {
      "record_id": "FUSED-2025-001",
      "timestamp": "2025-12-28T22:00:00Z",
      "type": "CBS_SIGNAL",
      "symptoms": ["diarrhea", "vomiting"],
      "verification_score": 1.0,
      "verified_by": ["CBS", "EMR"]
    },
    {
      "record_id": "EMR-2025-001",
      "timestamp": "2025-12-28T22:30:00Z",
      "type": "EMR_RECORD",
      "diagnosis": "Cholera",
      "treatment": "ORS + Antibiotics",
      "verification_score": 1.0
    }
  ],
  "golden_thread_summary": {
    "total_records": 2,
    "verified_records": 2,
    "verification_rate": 1.0
  }
}
```

## Mobile SDK integration

### Flutter SDK

```dart
import 'package:iluminara_sdk/iluminara_sdk.dart';

// Initialize SDK
final sdk = ILuminaraSDK(
  baseUrl: 'https://api.iluminara.health/v1',
  apiKey: 'your_api_key',
);

// Authenticate
await sdk.auth.login(
  username: 'chv_amina',
  password: 'secure_password',
  deviceId: 'DEVICE_001',
);

// Submit observation
final observation = Observation(
  observationId: 'OBS-2025-001',
  timestamp: DateTime.now(),
  location: Location(lat: 0.0512, lng: 40.3129),
  symptoms: [
    Symptom(name: 'diarrhea', severity: 8),
    Symptom(name: 'vomiting', severity: 7),
  ],
);

final result = await sdk.observations.submit(observation);
print('Verification score: ${result.verificationScore}');
```

## Offline-first architecture

### Queue management

```javascript
// Queue observation when offline
if (!navigator.onLine) {
  await sdk.queue.add(observation);
  console.log('Observation queued for sync');
}

// Auto-sync when connectivity returns
window.addEventListener('online', async () => {
  const syncResult = await sdk.queue.syncAll();
  console.log(`Synced ${syncResult.syncedCount} observations`);
});
```

### Local storage

```javascript
// Store observations locally
await sdk.storage.save('observations', observations);

// Retrieve from local storage
const cachedObservations = await sdk.storage.get('observations');
```

## Sovereignty compliance

All API operations enforce sovereignty constraints:

```python
from governance_kernel.vector_ledger import SovereignGuardrail

guardrail = SovereignGuardrail()

# Validate observation submission
guardrail.validate_action(
    action_type='Data_Processing',
    payload={
        'data_type': 'PHI',
        'processing_location': 'Edge_Node',
        'consent_token': observation.consent_token
    },
    jurisdiction='KDPA_KE'
)
```
