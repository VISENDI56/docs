## Overview

NVIDIA Omniverse Digital Twin provides real-time 3D visualization and simulation for "Camp-to-City" urban planning under the Shirika Plan. It enables zero-risk urban redevelopment by simulating infrastructure changes before physical construction.

<Info>
  Under the Shirika Plan, Dadaab and Kalobeyei are transitioning from refugee camps to integrated municipalities. Omniverse enables this transformation without displacing vulnerable populations.
</Info>

## Civilization-scale singularity

**The Context:** Under the Shirika Plan, camps are becoming towns. This requires massive urban planning (roads, sewage, power) without displacing vulnerable people.

**The Singularity:** Omniverse USD (Universal Scene Description) Pipeline.

**Deep Architecture:** A real-time, photorealistic 3D simulation of the entire settlement. Planners can "drag and drop" a new clinic or market in the digital twin to see how it affects foot traffic, airflow (disease spread), and flood risks before building it.

**Impact:** Zero-risk urban redevelopment.

## Quick start

```python
from nvidia_integrations.omniverse_digital_twin import OmniverseDigitalTwin

# Initialize digital twin
twin = OmniverseDigitalTwin(
    nucleus_server="omniverse://localhost",
    camp_name="Dadaab"
)

# Create camp layout
twin.create_camp_layout({
    "tents": 5000,
    "clinics": 3,
    "water_points": 10,
    "area_km2": 50
})

# Simulate disease spread
twin.simulate_disease_spread({
    "disease": "cholera",
    "initial_cases": 10,
    "r_effective": 2.3,
    "duration_days": 30
})

# Visualize resource allocation
twin.visualize_resource_allocation({
    "ors_packets": 10000,
    "antibiotics": 5000,
    "health_workers": 50
})
```

## Integration with Golden Thread

```python
from edge_node.sync_protocol.golden_thread import GoldenThread

gt = GoldenThread()

# Get verified outbreak data
fused_data = gt.fuse_data_streams(
    cbs_signal={"location": "Dadaab", "symptom": "diarrhea"},
    emr_record={"location": "Dadaab", "diagnosis": "cholera"}
)

# Update digital twin with verified data
twin.update_from_golden_thread(fused_data)
```
