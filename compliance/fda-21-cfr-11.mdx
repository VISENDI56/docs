## Overview

FDA 21 CFR Part 11 establishes criteria for electronic records and signatures to be equivalent to paper records and handwritten signatures. iLuminara implements **quantum-resistant timestamped audit trails (IP-09)** with FIPS 140-2 compliance, unique biometric signatures through Acorn Protocol, and immutable linkage between signatures and records.

## Key requirements

### §11.10: Controls for closed systems

<AccordionGroup>
  <Accordion title="(a) System validation">
    Validation ensuring accuracy, reliability, consistent intended performance, and ability to discern invalid or altered records
  </Accordion>
  <Accordion title="(e) Audit trails">
    Secure, computer-generated, time-stamped audit trails independently recording date/time of operator entries and actions
  </Accordion>
  <Accordion title="(g) Authority checks">
    Ensuring only authorized individuals can use system, sign records, access devices, alter records, or perform operations
  </Accordion>
</AccordionGroup>

### Acorn Protocol biometric signatures

```python
from governance_kernel.acorn_protocol import AcornBiometric

acorn = AcornBiometric()

# Somatic authentication: posture + location + stillness
signature = acorn.generate_signature(
    user_id="DR_AMINA_HASSAN",
    posture="seated_upright",
    location_gps=(0.0512, 40.3129),
    stillness_duration_seconds=3.2,
    action="APPROVE_CHOLERA_DIAGNOSIS"
)

# Unique to one individual, not reusable or reassignable (§11.100(a))
assert signature.is_unique()
assert not signature.is_reusable()

# Requires collaboration of at least two individuals for unauthorized use (§11.200(a)(3))
assert signature.requires_multi_party_compromise()
```

## Signature manifestations (§11.50)

Signed electronic records clearly indicate:

1. **Printed name of signer**: `DR_AMINA_HASSAN`
2. **Date and time**: `2025-12-25T17:00:00Z`
3. **Meaning**: `CLINICAL_APPROVAL_CHOLERA_DIAGNOSIS`

```python
# Generate signature manifestation
manifestation = audit.generate_signature_manifestation(
    signer="DR_AMINA_HASSAN",
    timestamp="2025-12-25T17:00:00Z",
    meaning="CLINICAL_APPROVAL_CHOLERA_DIAGNOSIS",
    record_id="DIAG_2025_001"
)

# Include in human-readable form
human_readable_with_signature = validator.generate_human_readable_copy(
    record_id="DIAG_2025_001",
    include_signature_manifestation=True
)
```

## Operational system checks (§11.10(f))

```python
from governance_kernel.operational_checks import SequenceEnforcer

enforcer = SequenceEnforcer()

# Define permitted sequence
enforcer.define_sequence(
    workflow="CHOLERA_DIAGNOSIS",
    steps=[
        "SYMPTOM_COLLECTION",
        "FRENASA_INFERENCE",
        "CLINICAL_REVIEW",
        "TREATMENT_APPROVAL",
        "MEDICATION_DISPENSING"
    ]
)

# Enforce sequencing
try:
    enforcer.execute_step(
        workflow="CHOLERA_DIAGNOSIS",
        step="MEDICATION_DISPENSING",
        current_state="SYMPTOM_COLLECTION"
    )
except SequenceViolationError as e:
    print(f"❌ Sequence violation: {e}")
    # Cannot dispense medication before clinical review
```
