## Eight conditions for lawful processing

### Condition 1: Accountability (ยง8)

Responsible party must ensure compliance with all conditions.

**iLuminara Implementation:**
```python
from governance_kernel.vector_ledger import SovereignGuardrail

# Information Officer designation
INFORMATION_OFFICER = "io@iluminara.health"

# Accountability enforcement
guardrail = SovereignGuardrail(
    jurisdiction='POPIA_ZA',
    enable_tamper_proof_audit=True
)
```

### Condition 2: Processing limitation (ยง9-12)

Process personal information lawfully, reasonably, and with consent.

**ยง11 - Consent:**
```python
# Explicit consent required for special personal information (health data)
guardrail.validate_action(
    action_type='Consent_Validation',
    payload={
        'consent_token': 'VALID_TOKEN',
        'consent_scope': 'treatment',
        'data_type': 'special_personal_information',  # Health data
        'consent_explicit': True
    },
    jurisdiction='POPIA_ZA'
)
```

**ยง12 - Justification:**
Processing without consent only if:
- Consent unreasonably difficult to obtain
- Used for historical, statistical, or research purposes
- Public body performing lawful function

Collect personal information for specific, explicitly defined, and lawful purpose.

### Condition 7: Security safeguards (ยง19)

Secure integrity and confidentiality of personal information.

**iLuminara Implementation:**
- **Encryption:** AES-256-GCM (Crypto Shredder)
- **Access control:** Role-based with unique user IDs
- **Audit:** Tamper-proof trail with Cloud Spanner
- **Transmission:** TLS 1.3 encryption

```python
from governance_kernel.crypto_shredder import CryptoShredder, SovereigntyZone

shredder = CryptoShredder(
    sovereignty_zone=SovereigntyZone.SOUTH_AFRICA,
    enable_audit=True
)

# Encrypt with ephemeral key
encrypted_data, key_id = shredder.encrypt_with_ephemeral_key(
    data=patient_record,
    retention_policy=RetentionPolicy.HOT,
    metadata={'jurisdiction': 'POPIA_ZA'}
)
```

### Condition 8: Data subject participation (ยง20-25)

Data subjects have rights to access, correct, and delete their information.

**ยง23 - Correction:**
```python
# API endpoint for correction requests
@app.route('/api/v1/correction-request', methods=['POST'])
def handle_correction_request():
    patient_id = request.json['patient_id']
    corrections = request.json['corrections']
    
    # Validate and apply corrections
    update_patient_record(patient_id, corrections)
    
    # Audit trail
    log_correction(patient_id, corrections)
```

**ยง24 - Deletion:**
```python
# Right to erasure via Crypto Shredder
def handle_deletion_request(patient_id):
    # Find all keys for patient
    keys = find_patient_keys(patient_id)
    
    # Shred all keys (data becomes irrecoverable)
    for key_id in keys:
        shredder.shred_key(key_id)
    
    # Audit trail
    log_deletion(patient_id)
```

## Cross-border information transfers (ยง72)

Transfer to third country only if:
1. Recipient country has adequate protection
2. Data subject consents
3. Transfer necessary for contract performance
4. Transfer in data subject's interest

**iLuminara enforcement:**
```yaml
# config/sovereign_guardrail.yaml
sovereignty:
  data_residency:
    allowed_zones:
      - "africa-south1"  # South Africa
    
    blocked_zones:
      - "us-*"           # Requires explicit consent
      - "asia-*"         # Sovereignty violation
  
  cross_border_transfers:
    popia_approved:
      - "EU"             # GDPR adequacy
      - "UK"             # UK GDPR
```

```python
# Block unauthorized transfers
guardrail.validate_action(
    action_type='Data_Transfer',
    payload={
        'data_type': 'special_personal_information',
        'destination': 'US_Cloud',
        'consent_token': None
    },
    jurisdiction='POPIA_ZA'
)
# Raises: SovereigntyViolationError (ยง72 violation)
```

## Special personal information (health data)

Health data is "special personal information" requiring heightened protection.

**ยง26 - Prohibition:**
Processing prohibited unless:
- Consent of data subject
- Necessary for establishing, exercising, or defending a right
- Necessary for compliance with legal obligation
- Necessary to protect legitimate interest of data subject
- Public body performing lawful function

**iLuminara implementation:**
```python
# Health data requires explicit consent
guardrail.validate_action(
    action_type='Data_Processing',
    payload={
        'data_type': 'special_personal_information',
        'data_category': 'health',
        'consent_token': 'EXPLICIT_CONSENT_TOKEN',
        'legal_basis': 'consent'
    },
    jurisdiction='POPIA_ZA'
)
```

## Breach notification

POPIA requires notification of breaches to Information Regulator and affected data subjects.

**iLuminara breach workflow:**
<Steps>
  <Step title="Detection">
    Security workflows detect unauthorized access
  </Step>
  <Step title="Assessment">
    Determine if breach affects special personal information
  </Step>
  <Step title="Notification">
    Notify Information Regulator and affected data subjects
  </Step>
  <Step title="Remediation">
    Crypto Shredder dissolves compromised data
  </Step>
  <Step title="Documentation">
    Maintain breach records for audit
  </Step>
</Steps>

```python
# Automatic breach notification
if breach_detected:
    notify_information_regulator()
    notify_affected_data_subjects()
    log_breach_record()
    
    # Dissolve compromised data
    shredder.shred_key(compromised_key_id)
```
