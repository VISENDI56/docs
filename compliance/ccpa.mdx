## Key consumer rights

### Right to know (¬ß1798.100)

Consumers have the right to know what personal information is collected, used, shared, or sold.

**iLuminara Implementation:**
```python
# API endpoint for data access requests
@app.route('/api/v1/ccpa/data-access', methods=['POST'])
def handle_data_access_request():
    consumer_id = request.json['consumer_id']
    verification_token = request.json['verification_token']
    
    # Verify consumer identity
    if not verify_consumer(consumer_id, verification_token):
        return {'error': 'verification_failed'}, 403
    
    # Return all personal information
    return {
        'categories_collected': get_categories_collected(consumer_id),
        'sources': get_data_sources(consumer_id),
        'business_purposes': get_business_purposes(consumer_id),
        'third_parties': get_third_party_disclosures(consumer_id),
        'personal_information': get_consumer_data(consumer_id)
    }
```

### Right to delete (¬ß1798.105)

Consumers have the right to request deletion of personal information.

**iLuminara Implementation:**
```python
from governance_kernel.crypto_shredder import CryptoShredder

# CCPA deletion via Crypto Shredder
@app.route('/api/v1/ccpa/delete', methods=['POST'])
def handle_deletion_request():
    consumer_id = request.json['consumer_id']
    verification_token = request.json['verification_token']
    
    # Verify consumer identity
    if not verify_consumer(consumer_id, verification_token):
        return {'error': 'verification_failed'}, 403
    
    # Find all keys for consumer
    keys = find_consumer_keys(consumer_id)
    
    # Shred all keys (data becomes irrecoverable)
    shredder = CryptoShredder()
    for key_id in keys:
        shredder.shred_key(key_id)
    
    # Audit trail
    log_deletion(consumer_id, 'CCPA_RIGHT_TO_DELETE')
    
    return {'status': 'deleted', 'consumer_id': consumer_id}
```

### Right to correct (¬ß1798.106) - CPRA

Consumers have the right to correct inaccurate personal information.

**iLuminara Implementation:**
```python
# API endpoint for correction requests
@app.route('/api/v1/ccpa/correct', methods=['POST'])
def handle_correction_request():
    consumer_id = request.json['consumer_id']
    corrections = request.json['corrections']
    verification_token = request.json['verification_token']
    
    # Verify consumer identity
    if not verify_consumer(consumer_id, verification_token):
        return {'error': 'verification_failed'}, 403
    
    # Apply corrections
    update_consumer_record(consumer_id, corrections)
    
    # Audit trail
    log_correction(consumer_id, corrections)
    
    return {'status': 'corrected', 'consumer_id': consumer_id}
```

## Consumer request verification

CCPA requires reasonable verification of consumer identity.

**iLuminara verification process:**
<Steps>
  <Step title="Request submission">
    Consumer submits request via web form or API
  </Step>
  <Step title="Identity verification">
    Two-factor authentication or government ID verification
  </Step>
  <Step title="Request processing">
    Process request within 45 days (extendable to 90 days)
  </Step>
  <Step title="Response delivery">
    Deliver response via secure portal or encrypted email
  </Step>
</Steps>

```python
def verify_consumer(consumer_id, verification_token):
    # Two-factor authentication
    if not validate_2fa_token(verification_token):
        return False
    
    # Match consumer ID with account
    if not consumer_exists(consumer_id):
        return False
    
    return True
```

## CPRA enhancements (effective 2023)

CPRA adds new requirements:

1. **California Privacy Protection Agency (CPPA)** - New enforcement authority
2. **Sensitive personal information** - Heightened protections for health data
3. **Right to correct** - New consumer right
4. **Automated decision-making** - Opt-out right for profiling
5. **Risk assessments** - Required for high-risk processing

**iLuminara CPRA compliance:**
