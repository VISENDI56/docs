## Overview

The **NVIDIA Omniverse Digital Twin** provides real-time 3D visualization and simulation of refugee camps, health facilities, and outbreak scenarios using Universal Scene Description (USD) and NVIDIA's Omniverse platform.

## Creating a digital twin

### Basic setup

```python
from nvidia_omniverse.digital_twin import RefugeeCampDigitalTwin

# Initialize digital twin
twin = RefugeeCampDigitalTwin(
    camp_name="Dadaab_Ifo",
    omniverse_server="omniverse://localhost"
)

# Initialize USD stage
twin.initialize_stage()
```

## Real-time outbreak visualization

### Update heatmap

```python
# Get case data from Golden Thread
from edge_node.sync_protocol.golden_thread import GoldenThread

gt = GoldenThread()
cases = gt.get_cases(
    start_date="2025-12-01",
    end_date="2025-12-31",
    disease="cholera"
)

# Convert to heatmap format
case_data = [
    {
        "x": case.location.x,
        "y": case.location.y,
        "severity": case.severity
    }
    for case in cases
]

# Update heatmap in digital twin
twin.update_outbreak_heatmap(
    case_data=case_data,
    disease="cholera"
)
```

## Integration with iLuminara stack

### Real-time data pipeline

```python
from google.cloud import pubsub_v1
import json

# Subscribe to outbreak alerts
subscriber = pubsub_v1.SubscriberClient()
subscription_path = subscriber.subscription_path(PROJECT_ID, "outbreak-alerts")

def callback(message):
    """Process outbreak alert and update digital twin"""
    data = json.loads(message.data)
    
    # Update heatmap
    twin.update_outbreak_heatmap(
        case_data=[{
            "x": data["location"]["x"],
            "y": data["location"]["y"],
            "severity": data["severity"]
        }],
        disease=data["disease"]
    )
    
    message.ack()

# Start listening
subscriber.subscribe(subscription_path, callback=callback)
```

## Water quality monitoring

### Real-time sensor integration

```python
# Update water source quality from IoT sensors
def update_water_quality(sensor_data):
    """Update water source visualization based on sensor readings"""
    
    for sensor in sensor_data:
        # Find corresponding water source
        water_source_id = sensor["source_id"]
        quality_score = sensor["quality_score"]
        
        # Update in digital twin
        twin.update_water_source_quality(
            source_id=water_source_id,
            quality_score=quality_score
        )

# Example sensor data
sensor_data = [
    {
        "source_id": "1",
        "quality_score": 0.8,
        "ph": 7.2,
        "turbidity": 2.5,
        "e_coli": 0
    },
    {
        "source_id": "2",
        "quality_score": 0.3,
        "ph": 6.8,
        "turbidity": 15.0,
        "e_coli": 150  # Contaminated
    }
]

update_water_quality(sensor_data)
```

## Compliance and sovereignty

### Data sovereignty

All digital twin data respects sovereignty constraints:

```python
from governance_kernel.vector_ledger import SovereignGuardrail

guardrail = SovereignGuardrail()

# Validate visualization data export
guardrail.validate_action(
    action_type='Data_Transfer',
    payload={
        'data_type': 'Geospatial_Health_Data',
        'destination': 'Omniverse_Server',
        'jurisdiction': 'KDPA_KE'
    },
    jurisdiction='KDPA_KE'
)
```
