## Quick deployment

### Method 1: Force sync script (recommended)

```bash
# Make script executable
chmod +x scripts/mintlify_force_sync.sh

# Run force sync
./scripts/mintlify_force_sync.sh

# Or specify branch
./scripts/mintlify_force_sync.sh global-health-singularity
```

This script:
1. Syncs latest changes from Git
2. Validates documentation structure
3. Deploys to Mintlify with force rebuild
4. Opens live documentation in browser

### Method 2: GitHub Actions

Push to main branch triggers automatic deployment:

```bash
git add .
git commit -m "docs: update documentation"
git push origin main
```

Monitor deployment: https://github.com/VISENDI56/docs/actions

### Method 3: Mintlify CLI

```bash
# Install Mintlify CLI
npm install -g mintlify@latest

# Local preview
mintlify dev

# Deploy to production
mintlify deploy --force
```

## GitHub Actions workflow

Automated deployment workflow: `.github/workflows/mintlify-deploy.yml`

### Triggers

- **Push to main/global-health-singularity** - Automatic deployment
- **Pull request** - Validation and preview
- **Manual dispatch** - Force rebuild option

### Jobs

<Steps>
  <Step title="Validate">
    - Validate docs.json syntax
    - Count documentation files
    - Check for broken links
    - Fortress status check
  </Step>
  <Step title="Deploy">
    - Install Mintlify CLI
    - Deploy to production
    - Generate deployment summary
  </Step>
  <Step title="Preview">
    - Generate preview for pull requests
    - Comment on PR with preview info
  </Step>
</Steps>

### Manual trigger

```bash
# Trigger workflow manually with force rebuild
gh workflow run mintlify-deploy.yml -f force_rebuild=true
```

## Validation

### Pre-deployment checks

```bash
# Validate docs.json
python3 -m json.tool docs.json

# Count MDX files
find . -name "*.mdx" | wc -l

# Check for broken links
grep -r "href=\"/" *.mdx
```

### Post-deployment verification

```bash
# Check site accessibility
curl -I https://visendi56.mintlify.app/

# Check specific page
curl -I https://visendi56.mintlify.app/quickstart
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Deployment fails">
    **Solution:**
    1. Check GitHub Actions logs
    2. Validate docs.json syntax
    3. Ensure all referenced MDX files exist
    4. Check MINTLIFY_API_KEY secret
  </Accordion>
  <Accordion title="Changes not reflected">
    **Solution:**
    1. Run force sync script
    2. Clear browser cache
    3. Wait 1-3 minutes for CDN propagation
    4. Check deployment status in GitHub Actions
  </Accordion>
  <Accordion title="docs.json validation error">
    **Solution:**
    1. Validate JSON syntax: `python3 -m json.tool docs.json`
    2. Check for trailing commas
    3. Ensure all page paths are correct
    4. Verify navigation structure
  </Accordion>
  <Accordion title="Missing pages">
    **Solution:**
    1. Check file paths in docs.json
    2. Ensure MDX files have frontmatter
    3. Verify file extensions (.mdx not .md)
    4. Check for typos in navigation paths
  </Accordion>
</AccordionGroup>
