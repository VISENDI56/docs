## Step 2: Check for broken links

Run Mintlify's diagnostic tool:

```bash
mint doctor
```

This checks for:
- ‚ùå Broken internal links
- ‚ùå Missing pages referenced in navigation
- ‚ùå Invalid MDX syntax
- ‚ùå Missing required frontmatter
- ‚ùå Orphaned pages (not in navigation)

### Fix common issues

**Broken link:**
```mdx
<!-- ‚ùå Wrong -->
[Link](/wrong-path)

<!-- ‚úÖ Correct -->
[Link](/correct-path)
```

**Missing frontmatter:**
```mdx
<!-- ‚ùå Wrong -->
# My Page

<!-- ‚úÖ Correct -->
---
title: My Page
description: Page description
---

# My Page
```

## Continuous deployment

### GitHub Actions workflow

The repository includes an automated deployment workflow at `.github/workflows/mintlify.yml`:

```yaml
name: "Mintlify Documentation Deployment"

on:
  push:
    branches: [ "main" ]
    paths:
      - 'docs/**'
      - '*.mdx'
      - 'docs.json'

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: npm install -g mintlify
      - run: mintlify doctor
      - run: mintlify build
      - run: mintlify update
        env:
          MINTLIFY_API_KEY: ${{ secrets.MINTLIFY_API_KEY }}
```

### Set up GitHub secret

1. Go to repository **Settings** ‚Üí **Secrets and variables** ‚Üí **Actions**
2. Click **New repository secret**
3. Name: `MINTLIFY_API_KEY`
4. Value: Your Mintlify API key
5. Click **Add secret**

### Automatic updates

Every push to `main` that modifies documentation files will:
1. ‚úÖ Validate with `mint doctor`
2. üèóÔ∏è Build optimized site
3. üöÄ Deploy to Mintlify
4. üîî Notify on success/failure

## Troubleshooting

<AccordionGroup>
  <Accordion title=\"Build fails with 'Module not found'\">
    **Solution:** Ensure all referenced pages exist and paths are correct in `docs.json`
    
    ```bash
    # Check for missing files
    mint doctor --check-links
    ```
  </Accordion>
  
  <Accordion title=\"Deployment fails with 'Unauthorized'\">
    **Solution:** Verify `MINTLIFY_API_KEY` is set correctly
    
    ```bash
    # Check environment variable
    echo $MINTLIFY_API_KEY
    
    # Re-export if needed
    export MINTLIFY_API_KEY="your_key"
    ```
  </Accordion>
  
  <Accordion title=\"Portal links return 404\">
    **Solution:** Ensure Streamlit apps are deployed and URLs are correct
    
    ```bash
    # Test each portal
    curl -I https://iluminara-war-room.streamlit.app
    curl -I https://iluminara-audit.streamlit.app
    ```
  </Accordion>
  
  <Accordion title=\"MCP queries not working\">
    **Solution:** Verify MCP configuration in `docs.json`
    
    ```json
    {
      \"mcp\": {
        \"enabled\": true,
        \"name\": \"iLuminara Sovereign Health Fortress\"
      }
    }
    ```
  </Accordion>
</AccordionGroup>

## Performance optimization

### Image optimization

```bash
# Optimize images before committing
npm install -g sharp-cli

# Resize and compress
sharp -i input.png -o output.png resize 800 --quality 80
```

### Code splitting

Mintlify automatically code-splits for optimal performance:
- Initial bundle: ~50KB gzipped
- Lazy-loaded pages: ~10KB each
- Search index: Loaded on demand

### CDN caching

Mintlify uses global CDN with:
- Edge caching for static assets
- Smart invalidation on updates
- 99.99% uptime SLA
