## Stage 1: Sensing

The **RegulatoryEntropySensor** detects compliance drift using **Kullback-Leibler (KL) Divergence**.

### How it works

```python
from governance_kernel.rco_engine import RegulatoryEntropySensor

sensor = RegulatoryEntropySensor(baseline_path="config/sectoral_laws.json")

# Measure drift
drift = sensor.measure_drift(
    data_stream={
        "data_residency_compliance": 0.85,  # Drift from 1.0
        "explainability_rate": 0.75,  # Significant drift
        # ... other metrics
    },
    framework_id="EU_AI_ACT"
)

print(f"Drift Score: {drift.drift_score:.4f}")  # KL Divergence
print(f"Drift Level: {drift.drift_level.value}")  # CRITICAL
print(f"Factors: {drift.contributing_factors}")
```

### Drift levels

| Level | KL Divergence | Description |
|-------|---------------|-------------|
| **NOMINAL** | < 0.05 | Normal operational variance |
| **ELEVATED** | 0.05 - 0.15 | Increased monitoring required |
| **CRITICAL** | 0.15 - 0.30 | Patch generation triggered |
| **CATASTROPHIC** | > 0.30 | Immediate intervention required |

## Stage 2: Prediction

The **LawEvolutionVector** forecasts regulatory amendments using **Monte Carlo simulations** on geopolitical metadata.

### How it works

```python
from governance_kernel.rco_engine import LawEvolutionVector, RegulatorySignal

oracle = LawEvolutionVector()

# Ingest external signal
signal = RegulatorySignal(
    signal_id="SIG_001",
    source="EU_AI_Act_Draft_Code_Dec2025",
    timestamp="2025-12-17T00:00:00Z",
    content="Draft Code of Practice released",
    impact_frameworks=["EU_AI_ACT", "GDPR"],
    confidence=0.87,
    metadata={"release_date": "2025-12-17"}
)
oracle.ingest_signal(signal)

# Predict amendment
prob, confidence, metadata = oracle.predict_amendment(
    framework_id="EU_AI_ACT",
    monte_carlo_iterations=10000
)

print(f"Amendment Probability: {prob:.2%}")  # 87%
print(f"Confidence: {confidence.value}")  # HIGH
```

### Confidence levels

| Level | Probability | Description |
|-------|-------------|-------------|
| **LOW** | < 50% | Unlikely to change |
| **MODERATE** | 50-70% | Possible amendment |
| **HIGH** | 70-90% | Likely amendment |
| **CERTAIN** | > 90% | Amendment imminent |

### Example prediction

```
"87% confidence that 'Pandemic Emergency' definition in IHR 2005 
will expand to include 'Climate-Induced Vectors' by Q3 2026."
```

## Stage 3: Synthesis

The **AutoPatchGenerator** writes law-as-code patches that tighten SovereignGuardrail thresholds automatically.

### How it works

```python
from governance_kernel.rco_engine import AutoPatchGenerator

generator = AutoPatchGenerator(laws_path="config/sectoral_laws.json")

# Generate hotfix
patch = generator.generate_hotfix(
    law_id="EU_AI_ACT",
    drift_score=0.18,  # CRITICAL
    drift_factors=["explainability_rate", "audit_coverage"],
    amendment_probability=0.87
)

print(f"Patch ID: {patch.patch_id}")
print(f"Type: {patch.patch_type}")  # THRESHOLD_TIGHTEN
print(f"Changes: {patch.changes}")
print(f"Rationale: {patch.rationale}")
```

### Patch types

| Type | Description | Example |
|------|-------------|---------|
| **THRESHOLD_TIGHTEN** | Reduce threshold values | Lower confidence threshold from 0.7 to 0.6 |
| **NEW_CONSTRAINT** | Add new compliance rule | Require enhanced monitoring |
| **SCOPE_EXPANSION** | Expand framework coverage | Add new data categories |
