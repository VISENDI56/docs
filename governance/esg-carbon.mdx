## ESG frameworks

### I. Carbon Border Adjustment Mechanism (CBAM)

**Status:** ‚úÖ Enforced  
**Region:** üá™üá∫ European Union  
**Scope:** Embedded emissions calculation

**Requirements:**
- Calculate embedded emissions per logistics hop
- Report emissions for goods entering EU
- Purchase CBAM certificates
- Quarterly reporting to national authorities

### II. Paris Agreement Article 6.2

**Status:** ‚úÖ Enforced  
**Region:** üåê Global  
**Scope:** Internationally Transferred Mitigation Outcomes (ITMOs)

**Requirements:**
- Prevent double counting of emissions reductions
- Corresponding adjustments between countries
- Transparent accounting
- Robust governance

### III. ICVCM Core Carbon Principles (CCP)

**Status:** ‚úÖ Enforced  
**Region:** üåê Global  
**Scope:** High-integrity carbon credits

**Requirements:**
- **Additionality:** Credits represent real emissions reductions
- **Permanence:** Reductions are long-lasting
- **Robust quantification:** Accurate measurement
- **No double counting:** Credits used only once

## CBAM compliance engine

### Calculate embedded emissions

```python
from governance_kernel.cbam_calculator import CBAMCalculator

calculator = CBAMCalculator()

# Calculate emissions for logistics chain
emissions = calculator.calculate_embedded_emissions(
    goods={
        "type": "medical_equipment",
        "weight_kg": 500,
        "origin": "Kenya",
        "destination": "Germany"
    },
    logistics_chain=[
        {
            "mode": "truck",
            "distance_km": 450,
            "fuel_type": "diesel"
        },
        {
            "mode": "ship",
            "distance_km": 6800,
            "fuel_type": "heavy_fuel_oil"
        },
        {
            "mode": "truck",
            "distance_km": 320,
            "fuel_type": "diesel"
        }
    ]
)

print(f"Total embedded emissions: {emissions['total_co2e_kg']:.2f} kg CO2e")
print(f"CBAM certificates required: {emissions['cbam_certificates']}")
```

### Emissions calculation methodology

**Formula:**
```
Embedded CO2e = Œ£ (Distance √ó Emission Factor √ó Weight)
```

**Emission factors (kg CO2e per tonne-km):**
- Truck (diesel): 0.062
- Ship (heavy fuel oil): 0.011
- Air freight: 0.602
- Rail: 0.022

## Paris Agreement Art. 6.2 compliance

### Prevent double counting with Chrono-Ledger

iLuminara uses **IP-09 Chrono-Audit** to prevent double counting of carbon credits:

```python
from governance_kernel.chrono_audit import ChronoLedger

ledger = ChronoLedger()

# Register carbon credit
credit_id = ledger.register_carbon_credit(
    project_id="KENYA_REFORESTATION_001",
    vintage_year=2024,
    co2e_tonnes=1000,
    issuing_country="Kenya",
    methodology="VCS_VM0042",
    verification_body="Verra"
)

# Transfer credit (with corresponding adjustment)
ledger.transfer_credit(
    credit_id=credit_id,
    from_country="Kenya",
    to_country="Germany",
    corresponding_adjustment={
        "kenya_ndc_adjustment": -1000,  # Kenya subtracts from NDC
        "germany_ndc_adjustment": +1000  # Germany adds to NDC
    }
)

# Retire credit (prevent reuse)
ledger.retire_credit(
    credit_id=credit_id,
    retiring_entity="German_Corporation_XYZ",
    retirement_reason="Voluntary offsetting"
)
```

### Corresponding adjustments

**Paris Agreement Art. 6.2 logic:**

```
IF Credit_Transferred(Country_A ‚Üí Country_B):
    Country_A.NDC -= Credit_Amount
    Country_B.NDC += Credit_Amount
    Credit.Status = TRANSFERRED
    
IF Credit_Retired(Country_B):
    Credit.Status = RETIRED
    Credit.Reusable = FALSE
```

## ICVCM Core Carbon Principles

### Verify carbon credit integrity

```python
from governance_kernel.icvcm_verifier import ICVCMVerifier

verifier = ICVCMVerifier()

# Verify credit against CCP
verification = verifier.verify_credit(
    credit_id="CREDIT_KE_001",
    project_details={
        "type": "reforestation",
        "location": "Kenya",
        "start_date": "2020-01-01",
        "methodology": "VCS_VM0042",
        "baseline_scenario": "No reforestation",
        "additionality_test": "Investment barrier test",
        "permanence_period": 40,  # years
        "monitoring_frequency": "annual",
        "verification_body": "Verra"
    }
)

if verification["compliant"]:
    print("‚úÖ Credit meets ICVCM Core Carbon Principles")
    print(f"   Additionality: {verification['additionality_score']}/100")
    print(f"   Permanence: {verification['permanence_score']}/100")
else:
    print("‚ùå Credit does not meet CCP")
    for issue in verification["issues"]:
        print(f"   - {issue}")
```

### CCP assessment criteria

<AccordionGroup>
  <Accordion title="1. Additionality">
    **Requirement:** Credits represent emissions reductions that would not have occurred without the project.
    
    **Tests:**
    - Investment barrier test
    - Common practice test
    - Regulatory surplus test
  </Accordion>
  
  <Accordion title="2. Permanence">
    **Requirement:** Emissions reductions are long-lasting and not easily reversed.
    
    **Requirements:**
    - Minimum permanence period (typically 40+ years for forestry)
    - Buffer pool for reversal risk
    - Monitoring and verification
  </Accordion>
  
  <Accordion title="3. Robust quantification">
    **Requirement:** Accurate measurement of emissions reductions.
    
    **Requirements:**
    - Conservative baseline scenario
    - Approved methodology
    - Independent verification
  </Accordion>
  
  <Accordion title="4. No double counting">
    **Requirement:** Credits are used only once.
    
    **Requirements:**
    - Unique serial number
    - Registry tracking
    - Corresponding adjustments (Art. 6.2)
  </Accordion>
</AccordionGroup>

## IP-09: Chrono-Audit

The Chrono-Audit ledger provides retroactive verification of carbon credits:

```python
from governance_kernel.chrono_audit import ChronoAudit

audit = ChronoAudit()

# Retroactive verification
verification = audit.verify_historical_credit(
    credit_id="CREDIT_KE_001",
    verification_date="2025-01-15",
    historical_data={
        "issuance_date": "2024-01-01",
        "transfers": [
            {"date": "2024-06-01", "from": "Kenya", "to": "Germany"},
        ],
        "retirements": [
            {"date": "2024-12-01", "entity": "German_Corp_XYZ"}
        ]
    }
)

# Check for double counting
if verification["double_counting_detected"]:
    print("‚ùå Double counting detected!")
    print(f"   Credit used {verification['usage_count']} times")
else:
    print("‚úÖ No double counting - Credit integrity verified")
```

### Chrono-Audit features

- **Immutable ledger:** SHA-256 hash chain prevents tampering
- **Retroactive verification:** Audit historical transactions
- **Cross-registry reconciliation:** Detect credits in multiple registries
- **Corresponding adjustment tracking:** Verify NDC adjustments
