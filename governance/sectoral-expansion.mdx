## Cross-sectoral validation

Many operations span multiple sectors. The Sectoral Compliance Engine automatically validates across all applicable sectors.

### Example: Health insurance payment

```python
# Payment for health services (Health + Finance)
result = engine.validate_cross_sectoral(
    action={
        \"type\": \"Insurance_Payment\",
        \"payment_method\": \"credit_card\",
        \"service_type\": \"surgery\",
        \"patient_id\": \"PAT_001\",
        \"provider_id\": \"HOSP_001\"
    },
    sectors=[Sector.HEALTH, Sector.FINANCE],
    jurisdiction=\"HIPAA_US\"
)

# Validates against:
# Health: HIPAA, HITECH
# Finance: PCI-DSS, SOX, GLBA
```

### Example: Student health records

```python
# School nurse accessing student health data (Education + Health)
result = engine.validate_cross_sectoral(
    action={
        \"type\": \"Record_Access\",
        \"record_type\": \"student_health\",
        \"accessor_role\": \"school_nurse\",
        \"student_age\": 15
    },
    sectors=[Sector.EDUCATION, Sector.HEALTH],
    jurisdiction=\"HIPAA_US\"
)

# Validates against:
# Education: FERPA, PPRA
# Health: HIPAA, state health privacy laws
```

### Example: Medical device supply chain

```python
# Manufacturing and shipping medical devices (Manufacturing + Transport + Health)
result = engine.validate_cross_sectoral(
    action={
        \"type\": \"Device_Shipment\",
        \"device_class\": \"IIb\",
        \"origin\": \"Germany\",
        \"destination\": \"Kenya\",

    jurisdiction=\"GDPR_EU\"
```

## Sectoral conflict resolution

When frameworks from different sectors conflict, QuantumNexus resolves them.

### Example: Data retention conflict

```python
from governance_kernel.quantum_nexus import QuantumNexus, ConflictResolutionStrategy

nexus = QuantumNexus()

# GDPR (Health): Minimize data retention
# SOX (Finance): 7 years retention for financial records
# Conflict: Health payment records

conflicts = nexus.detect_conflicts(
    frameworks=[\"GDPR\", \"SOX\", \"HIPAA\"],
    action={
        \"type\": \"Data_Retention\",
        \"data_type\": \"health_payment_record\",
        \"retention_days\": 2555  # 7 years
    },
    jurisdiction=\"GDPR_EU\"
)

resolution = nexus.resolve_conflicts(
    conflicts=conflicts,
    strategy=ConflictResolutionStrategy.BALANCED
)

# Resolution: 3 years retention
# - Satisfies SOX audit requirements
# - Respects GDPR minimization principle
# - Complies with HIPAA retention standards
```

## Sectoral API endpoints

The Compliance API provides sector-specific endpoints.

### Get sector frameworks

```bash
curl \"http://localhost:8081/api/compliance/frameworks?jurisdiction=KDPA_KE&sector=health\"
```

Response:
```json
{
  \"frameworks\": [
    \"KDPA\", \"GDPR\", \"HIPAA\", \"WHO IHR\", 
    \"Geneva Conventions\", \"AU Malabo Convention\"
  ],
  \"count\": 6,
  \"sector\": \"health\"
}
```

### Validate cross-sectoral action

```bash
curl -X POST http://localhost:8081/api/compliance/validate \\
  -H \"Content-Type: application/json\" \\
  -d '{
    \"action_type\": \"Payment_Processing\",
    \"payload\": {
      \"payment_method\": \"credit_card\",
      \"service_type\": \"medical\"
    },
    \"jurisdiction\": \"HIPAA_US\",
    \"sector\": \"finance\"
  }'
```

## Best practices

<AccordionGroup>
  <Accordion title=\"Specify primary sector\">
    Always specify the primary sector for your operation. The engine will automatically detect cross-sectoral requirements.
  </Accordion>
  
  <Accordion title=\"Use cross-sectoral validation explicitly\">
    For operations that clearly span multiple sectors, use `validate_cross_sectoral()` for better performance.
  </Accordion>
