## Quantum superposition logic

When frameworks conflict, the Nexus applies precedence rules:

<Steps>
  <Step title="Enforcement hierarchy">
    ABSOLUTE > STRICT > MODERATE > ADVISORY
  </Step>
  <Step title="Jurisdiction specificity">
    More specific jurisdiction > Global
  </Step>
  <Step title="Temporal precedence">
    Newer effective date > Older (if supersedes)
  </Step>
  <Step title="Restrictive interpretation">
    Most restrictive compliant path wins
  </Step>
</Steps>

## Basic usage

```python
from governance_kernel.quantum_law_nexus import QuantumLawNexus, LegalDomain

nexus = QuantumLawNexus()

# Validate a data transfer action
result = nexus.validate_action(
    action="Data_Transfer",
    jurisdiction="Kenya",
    domains=[LegalDomain.DATA_PROTECTION, LegalDomain.HEALTH_SECURITY],
    context={"data_type": "PHI", "destination": "EU"}
)

print(f"Governing Framework: {result['governing_framework']}")
print(f"Enforcement Level: {result['enforcement_level']}")
print(f"Rationale: {result['rationale']}")
```

## Dynamic Omni-Law Matrix

The Matrix provides real-time compliance orchestration:

```python
from governance_kernel.omni_law_matrix import OmniLawMatrix

matrix = OmniLawMatrix(enable_audit=True)

# Validate cross-border data transfer
result = matrix.validate_data_transfer(
    source_jurisdiction="Kenya",
    destination_jurisdiction="USA",
    data_type="PHI",
    consent_token=None,
    emergency_override=False
)

print(f"Status: {result.status.value}")

```
