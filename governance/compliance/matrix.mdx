## Framework categories

### Data protection frameworks

<AccordionGroup>
  <Accordion title="GDPR (EU) - Extraterritorial baseline">
    **Scope:** Any processing of EU residents' data
    
    **Key provisions:**
    - Art. 5: Six principles (lawfulness, purpose limitation, minimization, accuracy, storage limitation, integrity)
    - Art. 6: Lawful bases (consent, contract, legal obligation, vital interests, public interest, legitimate interests)
    - Art. 9: Special categories (health data requires explicit consent or public health exception)
    - Art. 22: Right to explanation for automated decisions
    - Art. 30: Records of processing activities
    - Art. 32: Security of processing
    - Chapter V: Cross-border transfers (SCCs, BCRs, adequacy decisions)
    
    **iLuminara implementation:**
    - SovereignGuardrail enforces all GDPR articles
    - SHAP explanations for Art. 22 compliance
    - Golden Thread maintains Art. 30 records

### Conflict resolution

When frameworks conflict, iLuminara applies the **most protective** standard:

```python
from governance_kernel.vector_ledger import SovereignGuardrail

guardrail = SovereignGuardrail()

# Multi-jurisdiction validation
result = guardrail.validate_multi_jurisdiction(
    action_type='Data_Transfer',
    payload={'data_type': 'PHI', 'destination': 'USA'},
    jurisdictions=['GDPR_EU', 'KDPA_KE', 'HIPAA_US']
)

# Applies strictest requirement:
# - GDPR: Requires SCCs
# - KDPA: Requires consent + proof of safeguards
# - HIPAA: Requires BAA
# Result: All three requirements must be met
```
