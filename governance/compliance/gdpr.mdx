## Core principles (Article 5)

### Lawfulness, fairness, and transparency

```python
from governance_kernel.vector_ledger import SovereignGuardrail

guardrail = SovereignGuardrail()

# All processing must have lawful basis
guardrail.validate_action(
    action_type='Data_Processing',
    payload={
        'lawful_basis': 'Art_6_1_e_public_interest',  # Public health surveillance
        'data_type': 'PHI',
        'purpose': 'outbreak_detection',
        'transparency_notice': 'provided_at_collection'
    },
    jurisdiction='GDPR_EU'
)
```

**Implementation:**
- Lawful basis documented for all processing activities
- Transparency notices provided at point of collection
- Privacy notices in local languages (Swahili, French, Portuguese)

### Purpose limitation

```python
# Purpose limitation enforced through data minimization
guardrail.validate_action(
    action_type='Data_Processing',
    payload={
        'original_purpose': 'cholera_surveillance',
        'new_purpose': 'malaria_research',  # Incompatible purpose
        'data_type': 'PHI'
    },
    jurisdiction='GDPR_EU'
)
# Raises: SovereigntyViolationError - Purpose incompatible with original consent
```

**Implementation:**
- Purpose tags attached to all data at collection
- Automated checks prevent purpose creep
- Re-consent required for incompatible purposes

### Data minimization

```python
from edge_node.frenasa_engine.voice_processor import VoiceProcessor

processor = VoiceProcessor(data_minimization=True)

# Only extract necessary fields
result = processor.process_voice(
    audio_data,
    extract_fields=['symptoms', 'location', 'timestamp'],
    exclude_fields=['name', 'id_number', 'phone']  # PII excluded
)
```

**Implementation:**
- Acorn Protocol limits PHI exposure
- Voice-to-JSON extracts only necessary fields
- Pseudonymization applied by default

### Accuracy

```python
from edge_node.sync_protocol.golden_thread import GoldenThread

gt = GoldenThread()

# Cross-source verification ensures accuracy
fused = gt.fuse_data_streams(
    cbs_signal={'symptom': 'fever', 'location': 'Dadaab'},
    emr_record={'diagnosis': 'malaria', 'location': 'Dadaab'},
    patient_id='PAT_001'
)

# Verification score: 1.0 (CONFIRMED) - Data accurate across sources
```

**Implementation:**
- Golden Thread cross-validates data from CBS, EMR, IDSR
- Automated correction of inaccurate data
- Data subject right to rectification (Art. 16)

### Storage limitation

```python
from governance_kernel.crypto_shredder import CryptoShredder, RetentionPolicy

shredder = CryptoShredder()

# Encrypt with retention policy
encrypted_data, key_id = shredder.encrypt_with_ephemeral_key(
    data=patient_record,
    retention_policy=RetentionPolicy.HOT,  # 180 days
    metadata={'jurisdiction': 'GDPR_EU'}
)

# Auto-shred after retention period
shredder.auto_shred_expired_keys()
```

**Implementation:**
- Retention policies: HOT (180d), WARM (365d), COLD (1825d)
- Crypto Shredder auto-shreds expired keys
- Data becomes cryptographically irrecoverable

## Article 9: Special categories (health data)

### Prohibition and exceptions

Health data processing is **prohibited by default** under Art. 9(1), but iLuminara operates under multiple exceptions:

```python
# Art. 9(2)(c) - Vital interests
guardrail.validate_action(
    action_type='High_Risk_Inference',
    payload={
        'data_type': 'health_data',
        'exception': 'Art_9_2_c_vital_interests',
        'justification': 'Patient unable to consent, life-threatening cholera',
        'consent_token': None  # Not required for vital interests
    },
    jurisdiction='GDPR_EU'
)

# Art. 9(2)(i) - Public health
guardrail.validate_action(
    action_type='Data_Processing',
    payload={
        'data_type': 'health_data',
        'exception': 'Art_9_2_i_public_health',
        'public_health_purpose': 'cross_border_threat_protection',
        'legal_basis': 'WHO_IHR_2005',
        'professional_secrecy': True  # Healthcare workers bound by secrecy
    },
    jurisdiction='GDPR_EU'
)
```
