## Sectoral Compliance Engine

Validates actions against sector-specific regulations.

### Supported sectors

| Sector | Frameworks | Description |
|--------|------------|-------------|
| **Health** | 14 | GDPR, HIPAA, KDPA, POPIA, HITECH, WHO IHR, Geneva Conventions |
| **Finance** | 8 | PCI-DSS, SOX, GLBA, MiFID II, Basel III, FATF, OFAC |
| **Education** | 3 | FERPA, COPPA, PPRA |
| **Energy** | 2 | NERC CIP, IEC 62443 |
| **Transport** | 3 | NHTSA, EASA, ICAO |
| **Agriculture** | 2 | FDA FSMA, EU CAP |
| **Manufacturing** | 2 | ISO 9001, IATF 16949 |
| **Telecommunications** | 3 | CALEA, ECPA, GDPR |

### Usage

```python
from governance_kernel.sectoral_compliance_engine import SectoralComplianceEngine, Sector

engine = SectoralComplianceEngine()

# Validate health sector action
result = engine.validate_action(
    action={
        \"type\": \"Data_Transfer\",
        \"data_type\": \"PHI\",
        \"destination\": \"EU_Cloud\"
    },
    sector=Sector.HEALTH,
    jurisdiction=\"KDPA_KE\"
)

if result.compliant:
    print(f\"✅ Compliant with {result.frameworks_applied}\")
else:
    print(f\"❌ Violations: {result.violations}\")
```

### Cross-sectoral validation

```python
# Validate action across multiple sectors
result = engine.validate_cross_sectoral(
    action={
        \"type\": \"Payment_Processing\",
        \"data_type\": \"PHI\",
        \"payment_method\": \"credit_card\"
    },
    sectors=[Sector.HEALTH, Sector.FINANCE],
    jurisdiction=\"HIPAA_US\"
)

# Result includes compliance status for each sector
for sector_result in result.sector_results:
    print(f\"{sector_result.sector}: {sector_result.compliant}\")
```

## QuantumNexus: Conflict Resolution

Resolves contradictory regulations using quantum superposition logic.

### The problem

Different frameworks often have conflicting requirements:

- **GDPR**: Requires data minimization
- **HIPAA**: Requires comprehensive audit logs
- **Conflict**: Minimization vs. comprehensive logging

### The solution

QuantumNexus resolves conflicts by:
1. Detecting contradictions between frameworks
2. Calculating conflict severity
3. Applying resolution strategy (strictest, permissive, balanced)
4. Generating compliant action recommendation

### Usage

```python
from governance_kernel.quantum_nexus import QuantumNexus, ConflictResolutionStrategy

nexus = QuantumNexus()

# Detect conflicts
conflicts = nexus.detect_conflicts(
    frameworks=[\"GDPR\", \"HIPAA\", \"KDPA\"],
    action={
        \"type\": \"Data_Retention\",
        \"retention_days\": 2555  # 7 years
    },
    jurisdiction=\"GDPR_EU\"
)

if conflicts:
    # Resolve using strictest interpretation
    resolution = nexus.resolve_conflicts(
        conflicts=conflicts,
        strategy=ConflictResolutionStrategy.STRICTEST
    )
    
    print(f\"Resolution: {resolution.recommended_action}\")
    print(f\"Explanation: {resolution.explanation}\")
```

### Resolution strategies

<Tabs>
  <Tab title=\"STRICTEST\">
    Apply the most restrictive requirement from all conflicting frameworks.
    
    **Use case:** Maximum compliance, risk-averse organizations
    
    **Example:** If GDPR requires 6 months retention and HIPAA requires 7 years, apply 6 months (strictest data minimization)
  </Tab>
  
  <Tab title=\"PERMISSIVE\">
    Apply the least restrictive requirement that satisfies all frameworks.
    
    **Use case:** Operational flexibility, innovation-focused
    
    **Example:** If GDPR allows 6 months and HIPAA requires 7 years, apply 7 years (satisfies both)
  </Tab>
  
  <Tab title=\"BALANCED\">
    Find middle ground that balances all requirements.
    
    **Use case:** Pragmatic compliance, balanced risk
    
    **Example:** Apply 3 years retention (between 6 months and 7 years)
  </Tab>
  
  <Tab title=\"JURISDICTION_PRIORITY\">
    Prioritize primary jurisdiction's requirements.
    
    **Use case:** Clear jurisdictional authority
    
    **Example:** If operating in Kenya, prioritize KDPA over GDPR
  </Tab>
</Tabs>

## ChronoAudit: Time-Travel Compliance

Validates compliance across time - retroactively and prospectively.

### Retroactive auditing

Validate historical actions against regulations that were in effect at the time.

```python
from governance_kernel.chrono_audit_engine import ChronoAuditEngine

engine = ChronoAuditEngine()

# Perform retroactive audit
report = engine.retroactive_audit(
    start_date=\"2024-01-01T00:00:00\",
    end_date=\"2025-12-31T23:59:59\",
    jurisdiction=\"KDPA_KE\"
)

print(f\"Compliance Rate: {report.compliance_rate:.1%}\")
print(f\"Violations: {report.non_compliant_events}\")
print(f\"Recommendations: {report.recommendations}\")
```

### Prospective auditing

Identify gaps before upcoming regulations take effect.

```python
# Check readiness for upcoming regulation
report = engine.prospective_audit(
    upcoming_regulation={
        \"name\": \"EU AI Act Amendment 2026\",
        \"required_fields\": [\"explainability\", \"bias_assessment\"],
        \"effective_date\": \"2026-01-01\"
    },
    effective_date=\"2026-01-01T00:00:00\",
    jurisdiction=\"GDPR_EU\"
)

print(f\"Days until compliance: {report.violations[0]['time_to_compliance']}\")
print(f\"Gaps identified: {len(report.violations)}\")
print(f\"Action plan: {report.recommendations}\")
```

### Temporal conflict detection

Detect when an action would violate regulations at different points in time.

```python
# Check if action complies across time
conflicts = engine.detect_temporal_conflicts(
    event=compliance_event,
    time_window_days=365
)

for conflict in conflicts:
    print(f\"Conflict at {conflict['timestamp']}: {conflict['frameworks']}\")
```

## Compliance API

REST API for real-time compliance validation and auditing.

### Start the API

```bash
python api/compliance_router.py
```

The API starts on `http://localhost:8081`

from governance_kernel.sovereign_guardrail_v3 import SovereignGuardrailV3
